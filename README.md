### 一、 设计理念与定位

- **核心定位：** 针对子女及家属，打造“安全监控+情感连接”的双重纽带 。
- **设计原则：** 极简主义交互。考虑到家属在紧急状态下的焦虑，UI 采用高对比度颜色和大字号，确保关键信息在一秒内触达 。
- **情感化叙事：** 拒绝罗列枯燥的传感器参数，通过 AI 大模型将数据转译为老人的生活状态描述（如“父亲今天心情不错”） 。

***

### 二、 软件功能架构

#### 1. “守护全景” (首页)

这是 App 的默认入口，提供父母当前状态的实时“生命体征”快照。

- **状态胶囊：** 顶部显著显示老人的佩戴状态、室内位置（基于 UWB 定位）及当前活跃度 。
- **环境看板：** 集成天气预报与室内环境数据，针对恶劣天气自动推送“减少外出”的提醒建议 。
- **动态时间轴：** 类似微信朋友圈的形式，记录老人一天的关键行为点（如：08:30 开始晨练、12:00 正常午餐） 。

#### 2. “紧急干预” (预警中心)

这是系统的核心，处理由硬件触发的所有异常信号 。

- **跌倒诊断专区：** 当手环与摄像头检测到跌倒，App 瞬间全屏红色预警，并展示 **AI 伤情报告**，包括着地姿态、疑似损伤部位及标准急救指引 。
- **防骗拦截站：** 当 VAD 硬件捕获疑似诈骗关键词（如“转账”、“安全账户”），同步推送通话摘要及拦截状态 。
- **三级梯度呼叫：** 预留一键联络村医、一键呼叫 120 及联系社区网格员的快速通道 。

#### 3. “时光机” (AI 综合报告)

利用云端 Qwen 大模型对长期数据的深度萃取，提供非线性的健康评估 。

- **心理人格画像：** 展示基于“月半技术”的情绪走向趋势图（短期、中期、长期异常判定） 。
- **步态退化预警：** 动态呈现老人过去一个月的步速、重心晃动规律，标记潜在的致跌风险点 。
- **AI 陪伴周报：** 每周生成一份充满温情的总结，指导子女本周该如何与老人沟通（如“本周老人叹气较多，建议周六打个长电话”） 。

***

### 三、 交互逻辑与技术集成

#### 1. 消息推送机制 (Push Notification)

- **分级触达：** 红色告警采用高权重通知，配合震动与音频；蓝色提醒（如低电量）采用静默消息。
- **双重验证：** 针对断网情况，后端 FastAPI 将自动切换为发送短信（SMS）至家属手机。

#### 2. 数据流转与接口设计

- **实时同步：** 利用 **WebSocket** 技术，确保手环检测到的跌倒信号能在 1 秒内推送到 App 界面 。
- **多模态视频窗：** 通过调用萤石开放平台 API，家属可以在收到报警的第一时间，直接在 App 内点击查看经过隐私保护处理的现场关键帧截图或回放 。

#### 3. 用户体验增强 (UX)

- **家庭共享模式：** 支持多个子女同时绑定一个老人账号，报警信息同步推送，并在处理时显示“二哥正在处理中”，避免重复救援 。
- **离家自动切换：** 当家属 App 检测到手环与网关断连，自动开启地图定位模式，并在首页显示老人的实时户外轨迹 。

***

### 四、 开发实施建议（面向全栈团队）

- **前端选型：** 建议使用 **Vue 3 + UniApp**，复用你已有的 Web 开发经验，快速打包为微信小程序。
- **后端对接：** 由 FastAPI 负责聚合所有硬件数据、萤石 API 返回值以及 AI 模型推理结果，通过统一的 RESTful 接口供 App 调用 。
- **适老化合规：** 在 App 显著位置提供隐私协议开关，明确说明“原始音频不存储，仅存储特征向量”，以满足官方合规性要求 。
